# 项目提交规范指南

## 提交流程（必须严格遵守）

### 第一步：暂存文件

```bash
git add .
```

### 第二步：运行规范提交命令

```bash
pnpm commit:zh
```

### 第三步：选择提交类型

- 根据修改内容选择合适类型：
  - `optimize`: 优化代码质量
  - `perf`: 性能提升
  - `fix`: 修复 Bug
  - `feat`: 新功能
  - `refactor`: 代码重构
  - 等等...

### 第四步：选择提交范围

- 选择受影响的模块或功能范围
- 可选择具体文件或目录

### 第五步：输入描述信息

- 清晰描述修改内容
- 遵循规范格式

### 第六步：等待代码校验

- 工具会自动运行 Prettier、ESLint 等校验
- **重要**：必须等待校验完成

### 第七步：检查校验结果

- 如果出现 `ELIFECYCLE Command failed with exit code` 错误
- 仔细查看完整错误信息，定位问题文件和行号
- 修复问题后重新暂存和提交

### 注意事项

- 严禁使用 `--no-verify` 绕过校验
- 每个步骤都必须完整执行
- 校验失败时必须修复问题才能继续
- 保持代码质量和规范一致性
